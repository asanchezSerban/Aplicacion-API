1. ¿Qué es Markdown y por qué lo usa Claude?

Markdown es un lenguaje de marcado ligero que usa símbolos de texto plano para dar formato. En vez de usar botones como en Word, escribes símbolos directamente en el texto y el lector los interpreta como formato visual.

Por ejemplo, rodear una palabra con dos asteriscos (**palabra**) la convierte en negrita. Escribir # antes de una línea la convierte en un título grande.

¿Por qué lo usa Claude?
Claude lee el fichero CLAUDE.md como sus instrucciones de proyecto. El formato Markdown le ayuda a entender la jerarquía de las instrucciones: qué es un título principal, qué es una subsección, qué es un elemento de lista, qué es código. Un CLAUDE.md bien estructurado con Markdown hace que Claude entienda mejor el proyecto y siga las instrucciones de forma más precisa.

2. Encabezados — # ## ###

Los encabezados se crean poniendo uno o más símbolos # al inicio de la línea, seguidos de un espacio. Cuantos más #, menor es el nivel del encabezado (más pequeño).

Lo que escribes Nivel Para qué se usa
# Título H1 — Título principal El nombre del documento o sección más importante. Solo uno por documento.
## Sección H2 — Sección Grandes bloques temáticos. En CLAUDE.md: ROL, STACK, BASE DE DATOS...
### Subsección H3 — Subsección Partes dentro de una sección. En CLAUDE.md: Backend, Frontend, dentro de Stack.
#### Apartado H4 — Apartado Detalle fino dentro de una subsección. Se usa poco.

Ejemplo real del CLAUDE.md:

Lo que escribes en el .md
## STACK TECNOLÓGICO

### Backend
- Runtime: .NET 10
- Framework: ASP.NET Core

### Frontend
- Framework: Angular 21
Cómo lo interpreta Claude
STACK TECNOLÓGICO ← sección principal │ ├── Backend ← subsección │ .NET 10 │ ASP.NET Core │ └── Frontend ← subsección Angular 21
Regla importante: Siempre deja un espacio entre el # y el texto. #Título no funciona. Debe ser # Título.

3. Separadores — ---

Tres guiones seguidos en una línea sola crean una línea horizontal divisoria. Se usa para separar secciones visualmente.

Lo que escribes
## Sección A
Contenido de A.

---

## Sección B
Contenido de B.
Resultado visual
Sección A Contenido de A. ───────────────────── Sección B Contenido de B.

También puedes usar *** o ___ (tres asteriscos o tres guiones bajos), los tres producen el mismo resultado.

Cuándo usarlo en CLAUDE.md: Para separar grandes bloques de instrucciones que son conceptualmente independientes entre sí. Por ejemplo, separar la sección de backend de la de frontend, o separar la especificación de la base de datos de los endpoints.

4. Texto en negrita, cursiva y tachado

Lo que escribes Resultado Símbolo usado Para qué sirve
**texto** texto Dos asteriscos por cada lado Énfasis fuerte, palabras clave, términos importantes
__texto__ texto Dos guiones bajos por cada lado Igual que ** (son equivalentes)
*texto* texto Un asterisco por cada lado Énfasis suave, términos técnicos, aclaraciones
_texto_ texto Un guion bajo por cada lado Igual que * (son equivalentes)
***texto*** texto Tres asteriscos por cada lado Negrita y cursiva a la vez. Para avisos críticos.
~~texto~~ texto Dos virgulillas por cada lado Texto tachado. Para indicar algo obsoleto o eliminado.

Ejemplo práctico:

Lo que **no debes hacer** es saltarte el contrato de la API.
Los *enums* se serializan como strings, ***nunca*** como números.
~~Usar NgModules~~ → usar Standalone Components.

5. Código — comillas invertidas

El símbolo de la comilla invertida es la tecla que en un teclado español está a la izquierda del 1: ` (también llamado backtick o acento grave).

Código en línea — una comilla invertida

Rodea texto con una sola comilla invertida para mostrar código dentro de una frase normal:

Lo que escribes
Usa `dotnet run` para arrancar.
El endpoint es `GET /api/clients`.
Resultado
Usa dotnet run para arrancar. El endpoint es GET /api/clients.

Bloque de código — tres comillas invertidas

Rodea varias líneas con tres comillas invertidas (```) al inicio y al final para crear un bloque de código. Opcionalmente puedes indicar el lenguaje justo después de las primeras tres comillas para que se coloree la sintaxis:

Lo que escribes
```csharp
public class Client
{
    public int Id { get; set; }
    public string Name { get; set; }
}
```
Resultado (bloque resaltado)
public class Client { public int Id { get; set; } public string Name { get; set; } }

Lenguajes que puedes indicar: csharp, javascript, typescript, sql, json, bash, html, css, python, entre muchos otros.

Muy importante para CLAUDE.md: Cuando le das a Claude fragmentos de código, DTOs, modelos o configuraciones dentro del CLAUDE.md, envuélvelos siempre en bloques de código con el lenguaje indicado. Claude reconoce mejor qué es código y qué son instrucciones en texto.

6. Listas — con guiones y con números

Lista sin orden (viñetas)

Usa un guion -, un asterisco * o un símbolo + seguido de un espacio. Los tres son equivalentes, pero el guion es el más común.

Lo que escribes
- PostgreSQL 15+
- .NET 10
- Angular 21
- Node.js 22
Resultado
• PostgreSQL 15+ • .NET 10 • Angular 21 • Node.js 22

Lista ordenada (numerada)

Escribe el número, un punto y un espacio. Puedes usar siempre 1. — Markdown numera automáticamente.

Lo que escribes
1. Arrancar PostgreSQL
2. Ejecutar dotnet run
3. Ejecutar npm start
4. Abrir el navegador
Resultado
1. Arrancar PostgreSQL 2. Ejecutar dotnet run 3. Ejecutar npm start 4. Abrir el navegador

Lista anidada (sublistas)

Indenta con 2 o 4 espacios para crear un subnivel:

Lo que escribes
- Backend
  - Controllers
  - Services
  - Models
- Frontend
  - Components
  - Services
Resultado
• Backend • Controllers • Services • Models • Frontend • Components • Services

7. Tablas

Las tablas se crean con barras verticales | y guiones -. La segunda fila (la de guiones) es obligatoria — separa los encabezados del contenido.

Lo que escribes
| Verbo  | Endpoint         | Código |
|--------|------------------|--------|
| GET    | /api/clients     | 200    |
| POST   | /api/clients     | 201    |
| PUT    | /api/clients/:id | 200    |
| DELETE | /api/clients/:id | 204    |
Resultado visual
VerboEndpointCódigo
GET/api/clients200
POST/api/clients201
PUT/api/clients/:id200
DELETE/api/clients/:id204

Puedes alinear el contenido de las columnas añadiendo dos puntos en la fila de guiones:

| Izquierda | Centro  | Derecha |
|:----------|:-------:|--------:|
| texto     | texto   | texto   |
  • :--- → alineado a la izquierda (por defecto)
  • :---: → centrado
  • ---: → alineado a la derecha

8. Citas — >

El símbolo > al inicio de una línea crea un bloque de cita (blockquote). Se usa para resaltar notas importantes, advertencias o información que debe destacar sobre el resto.

Lo que escribes
> IMPORTANTE: Los enums se
> serializan como strings,
> nunca como números enteros.
Resultado
▎ IMPORTANTE: Los enums se ▎ serializan como strings, ▎ nunca como números enteros.

Puedes anidar citas con >>:

> Nota principal
>> Detalle dentro de la nota
Uso recomendado en CLAUDE.md: Las citas son ideales para marcar las instrucciones no negociables o las advertencias críticas que Claude no debe ignorar. Por ejemplo:

> NOTA IMPORTANTE: No usar NgModules. Todos los componentes son standalone.

9. Enlaces e imágenes

Enlace

[texto visible](https://url-del-enlace.com)

Ejemplo: [Documentación de k6](https://k6.io/docs) → se ve como un enlace clicable con el texto "Documentación de k6".

Imagen

![texto alternativo](ruta/a/imagen.png)

Igual que un enlace pero con ! delante. El texto alternativo se muestra si la imagen no carga.

Enlace de referencia (para no repetir URLs)

Visita [nuestra API][api-ref] para más detalles.

[api-ref]: https://localhost:5000/swagger

Define el enlace una vez al final del documento y reutilízalo con [nombre][referencia].

10. Saltos de línea y párrafos

Lo que escribes Resultado
Una línea en blanco entre dos bloques de texto Crea un nuevo párrafo con separación visual
Dos espacios al final de una línea + Enter Salto de línea sin crear un párrafo nuevo
Un Enter simple (sin línea en blanco) En la mayoría de Markdown, se trata como el mismo párrafo
<br> Salto de línea forzado (HTML dentro de Markdown)
Error frecuente: Si escribes dos líneas seguidas sin línea en blanco entre ellas, Markdown las une en un mismo párrafo. Para separar visualmente dos párrafos, siempre deja una línea en blanco entre ellos.

12. Buenas prácticas para CLAUDE.md

Un CLAUDE.md bien estructurado le da a Claude más contexto y hace que sus respuestas sean más precisas. Estas son las recomendaciones específicas para este fichero:

Usa encabezados para separar secciones grandes

Claude procesa el CLAUDE.md como instrucciones jerárquicas. Los encabezados le indican qué pertenece a qué sección:

## ROL Y OBJETIVO
## STACK TECNOLÓGICO
## BASE DE DATOS
## BACKEND
## FRONTEND

Usa código para todo lo que sea código

Cuando pongas nombres de clases, rutas, comandos, fragmentos de código o configuraciones, usa siempre las comillas invertidas o los bloques de código. Así Claude distingue claramente las instrucciones del código que debe generar:

## Bueno ✓
El modelo se llama `Client` y tiene el campo `Status` de tipo `ClientStatus`.
La ruta es `GET /api/clients/{id}`.

## Malo ✗
El modelo se llama Client y tiene el campo Status de tipo ClientStatus.

Usa listas para enumeraciones

En vez de escribir varias cosas en el mismo párrafo, ponlas en lista. Claude las procesa mejor y es menos probable que olvide algún elemento:

## Bueno ✓
Formatos permitidos para el logo:
- .jpg
- .jpeg
- .png
- .gif
- .webp

## Malo ✗
Los formatos permitidos para el logo son jpg, jpeg, png, gif y webp.

Marca lo no negociable con negrita o citas

Usa **texto** para instrucciones críticas que Claude no debe ignorar, y > para advertencias destacadas:

**Los enums se serializan como strings en JSON, nunca como enteros.**

> IMPORTANTE: No usar NgModules. Todos los componentes son standalone
> con imports explícitos.

Usa tablas para el contrato de la API

Las tablas son la forma más clara de documentar endpoints, campos de BD o DTOs:

| Verbo  | Ruta                    | Body      | Respuesta |
|--------|-------------------------|-----------|-----------|
| GET    | /api/clients            | —         | 200       |
| POST   | /api/clients            | FormData  | 201       |
| PATCH  | /api/clients/{id}/status| JSON      | 200       |

Usa separadores --- entre secciones independientes

Ayudan a Claude a entender los límites entre bloques de instrucciones completamente distintos:

## BACKEND — DETALLE COMPLETO
... instrucciones de backend ...

---

## FRONTEND — DETALLE COMPLETO
... instrucciones de frontend ...
Resumen final

Los tres símbolos más importantes para un CLAUDE.md son:

# / ## / ### — para crear jerarquía y que Claude sepa de qué trata cada sección.
``` — para envolver todo el código y que Claude lo distinga de las instrucciones.
- — para listar elementos de forma que Claude los procese uno a uno sin perderse ninguno.

Con solo estos tres bien aplicados, la calidad de las respuestas de Claude mejora notablemente.
